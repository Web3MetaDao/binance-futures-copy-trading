# 更新日志 (Changelog)

本文档记录了币安跟单系统从初始版本到 Pro V2 版本的完整演进历史。

## [Pro V2] - 2026-04-28 (进阶升级版)

### 🚀 新增特性 (Features)
- **WebSocket 实时推送**：接入 Binance User Data Stream，将跟单延迟从 30 秒轮询降至毫秒级。
- **Prometheus 监控**：新增 `/metrics` 端点，支持 Grafana 实时监控胜率、夏普率、API 延迟等 12 项专业指标。
- **健康检查与告警**：后台线程实时监控心跳、余额、熔断状态，异常时自动通过 Telegram 推送告警。
- **MTF 多时间框架过滤**：15m 信号必须与 1h/4h 趋势一致才执行，大幅降低逆势开仓概率。
- **市场状态分类器**：基于 ADX 和布林带宽度，自动识别趋势（全仓）、震荡（减仓 50%）和高波动（跳过）市场。
- **动态交易员权重**：根据交易员过去 30 天的胜率、盈亏比和最大回撤，自动计算 Softmax 权重，淘汰表现差的信号源。
- **Kelly Criterion 动态仓位**：根据历史胜率和盈亏比自动计算最优仓位比例（半 Kelly），最大化长期资金增长率。
- **ATR 追踪止损**：盈利超过 1R 后，止损线自动随价格上移，锁定利润的同时让利润奔跑。
- **分批建仓与止盈**：高分信号分两批建仓（突破+回调），盈利 1R 时平仓 50%，剩余 50% 追踪止损。
- **相关性过滤**：自动识别高度相关的资产（如 BTC/ETH，SOL/AVAX），避免同赛道风险过度集中。
- **Redis 消息队列**：引入 Pub/Sub 解耦信号抓取与执行，支持横向扩展（无 Redis 时自动降级为内存队列）。
- **SQLite 数据库持久化**：替代 JSONL 文件，支持复杂查询、权益曲线记录和多账户隔离。
- **多账户支持**：支持配置多个 Binance 子账户，每个账户独立的资金比例、风控参数和跟单目标。
- **Vectorbt 高级回测**：纯 Numpy 向量化回测引擎，支持参数网格搜索，自动生成包含夏普率、卡玛比率和月度收益热力图的 HTML 报告。

### 🐛 修复与优化 (Fixes & Optimizations)
- **[最高标准审计]** 修复了 WebSocket 多线程读写竞态、资源泄漏、回调异常崩溃问题。
- **[最高标准审计]** 修复了 Redis 降级后无法自动恢复的问题，增加了 `xreadgroup` 空值校验与批量发布原子性保障。
- **[最高标准审计]** 区分了 SQLite 读写模式的 `commit`，为所有读操作增加了锁保护，并启用了 `trader_stats_cache` 缓存表。
- **[最高标准审计]** 修复了 K 线数据不足时的边界异常，增加了 MTF（多时间框架）API 调用的本地缓存。
- **[最高标准审计]** 修正了 Sharpe 比率的年化计算错误，以及非复利模式下的资金曲线累加错误。

## [Pro V1] - 2026-04-28 (专业版)

### 🚀 新增特性 (Features)
- **多交易员支持**：支持同时配置多个交易员链接，按权重聚合信号，并内置冲突仲裁机制。
- **自动止损止盈**：开仓后自动挂出 `STOP_MARKET` 和 `TAKE_PROFIT_MARKET` 订单。
- **最大回撤熔断**：当账户回撤超过设定阈值时，自动暂停开仓并平掉所有仓位。
- **Smart Money 评分**：基于资金费率、OI 变化、多空比、ATR 和成交量 5 个维度对信号进行打分。
- **软降杠杆**：根据评分结果动态调整仓位（优秀100%/一般50%/差暂停）。
- **真复利模式**：每笔平仓后立即拉取最新余额，直接作为下一笔基础资金。
- **Telegram 通知**：实时推送开平仓、止损止盈、熔断告警及每日收益汇总。
- **Web 控制面板**：基于 Flask 的可视化面板，支持实时查看仓位、收益曲线和修改配置。
- **一键部署**：提供 Docker Compose 和 systemd 两种生产级部署方案。
- **备用信号链路**：主链路（公开实盘）失败时，自动切换到 Binance Smart Money 排行榜抓取头部交易员持仓。

### 🐛 修复与优化 (Fixes & Optimizations)
- **[安全审计]** 修复了平仓后名义仓位未及时更新的问题，避免了超额开仓和漏开仓。
- **[安全审计]** 引入了原子写入（`os.replace`）和文件锁，彻底杜绝了进程崩溃导致的状态文件损坏。
- **[安全审计]** 优化了日志读取的内存占用，并引入了日志轮转限制磁盘占用。
- **[安全审计]** 锁定了 `requirements.txt` 中所有核心依赖的主版本号，防止供应链投毒。
- **[完整性审查]** 修复了 `trade_log` 写入时若日志目录不存在会静默失败的问题。

## [V0] - 2026-04-28 (初始原型)

### 🚀 初始特性 (Initial)
- **基础骨架**：搭建了"公开实盘 → 自动跟单下单"的 Python 可运行骨架。
- **单交易员抓取**：支持解析 Binance App 分享链接（如 `uni-qr/cpro`）并抓取持仓。
- **基础执行引擎**：支持按固定比例（`CAPITAL_RATIO`）计算仓位并下单。
- **基础回测**：提供简单的 CSV 回测脚本。
