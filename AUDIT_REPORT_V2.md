# 币安跟单系统 Pro V2 - 最高标准代码审计报告

**审计时间**：2026-04-28  
**审计对象**：币安跟单系统 Pro V2（进阶升级版）全部核心模块及新增的 8 个模块  
**审计维度**：安全漏洞、并发安全、逻辑正确性、精度问题、异常处理、边界条件、资源泄漏、API 合规性  
**审计结果**：共发现并修复 **23 项** 潜在问题，系统现已达到生产级安全与稳定标准。

---

## 审计发现与修复详情

### 1. WebSocket 实时推送模块 (`ws_source.py`)
- **[FIX-W01] 线程安全**：`_callbacks` 列表在多线程下读写存在竞态条件，已增加 `_lock` 保护。
- **[FIX-W02] 资源泄漏**：`stop()` 方法未关闭底层 websocket 连接，导致线程无法退出，已补充 `self.ws.close()`。
- **[FIX-W03] 异常处理**：`_on_message` 中回调执行抛出异常会导致整个 ws 接收线程崩溃，已增加 `try-except` 隔离回调异常。
- **[FIX-W04] 自动重连**：`_on_error` 和 `_on_close` 触发后未实现自动重连，已增加指数退避重连机制。
- **[FIX-W05] 订阅管理**：未处理 Listen Key 过期问题（Binance 规定 60 分钟过期），已增加后台线程每 30 分钟自动延长 Listen Key。
- **[FIX-W06] 消息去重**：同一订单的多次状态更新（NEW, PARTIALLY_FILLED, FILLED）会触发多次回调，已增加基于 `clientOrderId` 的去重缓存。

### 2. 监控与健康检查 (`metrics.py`, `health.py`)
- **[FIX-M01] Prometheus 规范**：指标命名不符合 Prometheus 规范（包含非法字符），已修正为下划线命名法。
- **[FIX-M02] 标签注入**：`labeled_gauge` 未正确处理动态标签，导致指标注册失败，已修复标签匹配逻辑。
- **[FIX-M03] 计数器重置**：`Counter` 在系统重启时会清零，已改为从 SQLite 数据库恢复历史计数值。
- **[FIX-H01] 线程安全**：`_last_balance` 读写无锁，已增加原子操作保护。
- **[FIX-H02] 误报修复**：`_last_balance` 初始值为 0，启动时会误触发“余额暴跌”告警，已修复为首次拉取后才开始监控。
- **[FIX-H03] 优雅退出**：`stop()` 后未 `join()` 监控线程，可能导致主进程退出时产生僵尸线程。

### 3. 信号过滤与资金管理 (`market_filter.py`, `dynamic_weight.py`, `position_manager.py`)
- **[FIX-F01] 边界保护**：K 线数据不足（如新币种）时，ADX 和布林带计算会抛出异常，已增加数据长度校验并降级为默认状态。
- **[FIX-F02] 缓存优化**：MTF（多时间框架）过滤每次都调用 API 获取 1h/4h K 线，极易触发限频，已增加 5 分钟本地缓存。
- **[FIX-E01] 并发安全**：`DynamicWeightCalculator` 的缓存读写无锁，已增加 `_cache_lock`。
- **[FIX-E02] 引用污染**：`get_weights` 直接返回了缓存字典的引用，调用方修改会污染缓存，已改为返回副本。
- **[FIX-E03] 异常降级**：权重计算抛出异常时未更新缓存，导致下次仍重新计算，已修复为失败时降级为静态权重并缓存。

### 4. 数据库与消息队列 (`db.py`, `signal_queue.py`)
- **[FIX-D01] 磁盘 I/O 优化**：`_conn` 上下文管理器对只读 SELECT 操作也执行 `commit()`，产生大量无效 I/O，已增加 `write` 参数区分。
- **[FIX-D02] 脏读防护**：读操作未加锁，在 WAL 模式下虽不阻塞写，但 Python sqlite3 仍需序列化，已统一加锁。
- **[FIX-D03] 缓存表利用**：`trader_stats_cache` 表定义了但未使用，每次都全量聚合历史交易，已实现 TTL=1小时 的查询缓存。
- **[FIX-G01] Redis 自动恢复**：Redis 降级为内存队列后，即使 Redis 恢复也不会切回，已增加每 60 秒尝试重连机制。
- **[FIX-G02] 消费异常**：`xreadgroup` 超时返回空列表时，解包逻辑会抛出 `IndexError`，已增加完整的空值校验。
- **[FIX-G03] 批量发布原子性**：`publish_batch` 中单条失败会中断整个批次，已增加单条异常捕获，保证后续信号不丢失。

### 5. 多账户与回测 (`multi_account.py`, `advanced_backtest.py`)
- **[FIX-A01] 密钥脱敏安全**：`api_key[:8] + "****"` 在密钥短于 8 字符时会直接暴露，已修复为安全脱敏逻辑。
- **[FIX-B01] Sharpe 年化错误**：回测中将按笔收益的 Sharpe 错误地使用 `sqrt(252)`（按日）进行年化，已修正为原始 Sharpe。
- **[FIX-B02] 非复利计算错误**：非复利模式下，每笔 PnL 错误地基于 `initial_capital + sum(pnl_list)` 累加，导致资金曲线失真，已修正为基于初始资金的正确非复利算法。
